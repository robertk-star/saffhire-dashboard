app/tazworks/orders/[orderGuid]/searches/[searchGuid]/page.tsx

What changed:
- Removed the page-level fallback to process.env.TAZWORKS_CLIENT_GUID.
- The search detail page now requires and uses the actual clientGuid passed in the URL.
- The National Crim page now uses the existing lib/tazworks.ts getAllTazworksSearchResults helper.
- Added a warning if the page is opened without a clientGuid.

Why:
- The proxy default client GUID is only a fallback.
- For multiple TazWorks clients, each order/search page must pass the selected order's actual clientGuid.
- If the wrong clientGuid is used, All Search Results will not return alias/address data.
